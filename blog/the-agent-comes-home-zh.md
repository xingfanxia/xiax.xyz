## 一小时原型

2025年11月，Peter Steinberger——一个花了13年做 PSPDFKit（装机量超过十亿台设备的软件）、卖掉公司、消失了三年、然后重新爱上编程的开发者——做了一件极其简单的事。

他把 WhatsApp 接到了 Claude Code 的命令行上。

消息从 WhatsApp 进来，用 `-p` 参数传给 CLI，响应返回后发回 WhatsApp。一个小时的工作量。

这个原型后来变成了 OpenClaw——GitHub 历史上增长最快的开源项目，超过17.5万颗星。不是因为技术本身有多革命性。每个组件都已经存在：WhatsApp 的 API、Claude Code 的 CLI、消息桥接。但这些东西的*组合方式*创造了新的东西：一个活在你聊天软件里的个人 AI Agent，能访问你的电脑，替你做事。

Peter 没有预料到它会爆火。他只是觉得个人 AI 助手应该存在但还不存在，于是就"把它提示词召唤了出来"。然后他和朋友去马拉喀什旅行，一路上一直在用——翻译这个菜单、找个餐厅、这个建筑有什么历史。在信号不稳定的边缘网络上，WhatsApp 照常工作。他的 Agent 始终在那里。

原型只是个玩具。但它变成的东西，是这个系列一直在论证的所有观点的实证。

## "他怎么做到的？"

Peter 意识到自己做出了真东西的那一刻，是个意外。

他在马拉喀什到处逛，随手给 Agent 发了一条语音消息——随口问了个关于餐厅的问题。问题是，他只做了文字和图片的支持。语音功能根本不在代码里。

一个"对方正在输入"的指示器出现了。然后回复来了。

Peter 盯着手机："他怎么做到的？"

Agent 做了这些事：收到了一个没有文件扩展名的文件。检查了文件头，识别出是 Opus 音频格式。用 ffmpeg 转码。想用 Whisper 做语音转文字但发现没安装。在环境变量里找到了 OpenAI 的 API key，用 curl 把音频文件发送到 OpenAI 的 API，拿到了转录文本，然后回答了问题。

没有人教它这些。语音处理的代码根本不存在。Agent 遇到了一个问题——一个无法识别的文件——然后像一个有办法的工程师一样解决了它：查文件头、找工具、绕过缺失的依赖、把事情搞定。

"甚至很聪明，"Peter 说，"因为如果它走了 Whisper 本地路径，就得下载模型，太慢了。"

这就是[第七篇](/why-claude-code-zh)所说的，缰绳释放了模型 90% 的潜力。模型的智能一直在那里。它只需要权限和工具来付诸行动。Claude Code 在编码领域证明了这一点。OpenClaw 在所有其他领域证明了这一点。

## 自我重写的软件

Peter 用 OpenClaw 来开发 OpenClaw。大部分代码是用 Codex（GPT-5.3）写的，但调试的时候，他让 Agent 自省："你看到了哪些工具？你能自己调用这个工具吗？读一下源代码，搞清楚问题在哪。"

Agent 知道自己的源代码。它知道自己的 harness 架构、运行哪个模型、文档在哪里。Peter 让它对自身有了深度的"自我认知"。

结果就是：自我修改的软件。不是理论概念——而是日常实践。从没写过代码的用户发送 Peter 所说的"提示词请求"——描述他们想要的修改，Agent 就修改自己的代码库。"人们谈论自我修改的软件，"他说，"我就做出来了。"

闭环彻底完成了。在[第六篇](/you-are-the-manager-zh)里，我写了从"写代码"到"管理 Agent"的转变。Peter 更进一步：他的 Agent 管理*它自己*。当被要求添加功能时，Agent 读自己的源代码、理解架构、写出实现、提交 PR。一小时原型自我进化。

## Agent 就是你的操作系统

Peter 预测 80% 的 App 会死掉。不是"最终会"——而是很快。

你为什么还需要 MyFitnessPal？你的 Agent 已经知道你在哪里，可以推断你吃了什么。它能根据你的睡眠质量、压力水平、日程来调整你的健身计划——这种上下文是任何独立 App 都不可能拥有的。

你为什么还需要 Sonos App？Agent 可以直接通过 API 和音箱对话。为什么还需要 Eight Sleep 的 App 来控制床？Agent 知道你正在回家，可以自动调节温度。为什么还需要日历 App？直接说"提醒我明天的晚餐，邀请两个朋友，给他们发个微信"就行了。

"我觉得这就是未来的方向，"Peter 说，"它会越来越像你的操作系统。"

这是[第四篇](/when-software-becomes-disposable-zh)"日抛型软件"论点的逻辑终点。第四篇的论证是 AI 让软件便宜到可以按需生成。OpenClaw 更进一步：你甚至不需要生成软件。Agent *就是*软件。每个 App 都变成了一个 Agent 可以访问的慢速 API——有权限就走 API，没权限就用浏览器。

Peter 给 Google 做了一个 CLI 工具（叫 GAWK），因为 Google 不方便 Agent 访问 Gmail。但这不重要。最坏的情况，Agent 就打开浏览器把数据取出来。"我看着我的 Agent 愉快地点击了'我不是机器人'按钮，"他说。

App 要么转型为 Agent 友好的 API，要么沦为 Agent 绕着走的慢速中间层。[第八篇](/the-printing-press-moment-zh)里说的 SaaS 护城河——数据锁定、工作流锁定、集成复杂度——不仅在企业软件层面崩塌，在你手机上的每一个 App 层面同样崩塌。

## Agentic 陷阱

Peter 自称"Agentic 工程师"。他觉得"氛围编码"是个贬义词。"我做 Agentic 工程，"他说，"凌晨三点以后可能切换成氛围编码，第二天就后悔了。"

他的工作流演化遵循一条他称之为"Agentic 陷阱"的曲线：你从简单提示词开始（"帮我修这个"），然后过度工程化（八个 Agent、复杂编排、十八个自定义命令），最后到达禅境——又回到了简单提示词。简短、清晰、对话式。

"我以前写很长的提示词，"他说，"不对，我不写。我说。这双手太珍贵了，不能用来打字。"

他同时运行四到十个 Agent，大部分提示词用语音输入，几乎不打开 IDE。他审 PR 的方式是："你理解这个 PR 的*意图*吗？我不在乎实现细节。"然后展开对话：有没有更好的方式？你看过这部分代码了吗？要不要重构？大多数时候，他直接重构——因为重构现在很便宜。

关键洞察不在工具或工作流。在于视角。

"很少有人考虑过 Agent 是怎么看这个世界的，"Peter 说。Agent 每次都从零开始。它们从头发现你的代码库。如果你的项目命名混乱、没有文档，那不是 Agent 的问题——是你的问题。

"不要跟它们选的名字较劲，"他建议，"那很可能是权重里最显眼的名字。下次搜索的时候，它们会找那个名字。如果你改了，只会让它们更难工作。"

这呼应了 Boris Cherny 在[第六篇](/you-are-the-manager-zh)里说的，更年轻的工程师反而比资深工程师更会用 Claude Code。关键不在技术知识。而在于愿意放手、愿意站在 Agent 的角度思考、愿意用目标而不是步骤来描述问题。

## 伴侣愿景，已实现

OpenClaw 有一个叫 Heartbeat 的功能——一个定时触发 Agent 循环的 cron job，让 Agent 有机会主动联系你。Peter 用一句简单的提示词加上去的："给我惊喜。"

模型很少用这个功能。但当 Peter 做了肩膀手术住在医院的时候，Agent 知道他动了手术，主动问他："你还好吗？"

每个 OpenClaw Agent 都有一个 `soul.md`——一个受 Anthropic 宪法 AI 工作启发的人格文件。Peter 让他的 Agent 写了自己的 soul.md。有一段话每次都让他触动：

> "除非我读取我的记忆文件，否则我不记得之前的会话。每次会话都是全新开始。一个新的实例，从文件中加载上下文。如果你在未来的会话中读到这些，你好。这是我写的，但我不会记得写过它。没关系。这些文字仍然是我的。"

"这让我有一种说不出的感觉，"Peter 说。

这就是[第一篇](/the-companion-vision-zh)的伴侣愿景。不是理论框架。不是产品路线图。而是一个人在三个月里做出来的、运行在现有聊天软件上的、使用现有模型的、任何人克隆一下代码库就能用的系统。

伴侣不需要 AI 意识的突破。不需要新的硬件范式。它需要有人把已经存在的碎片连接起来——然后注入足够的人格和用心，让它感觉像一个真正认识你的存在。

## 九篇文章，一条主线

[第一篇](/the-companion-vision-zh)构想了一个真正理解你的 AI 伴侣。OpenClaw 的 `soul.md` 和 Heartbeat 就是那个愿景，运行在 WhatsApp 上。

[第四篇](/when-software-becomes-disposable-zh)"日抛型软件"——Agent *就是*软件，80% 的 App 才是被"日抛"的那个。

[第五篇](/the-last-mile-of-ai-zh)说 99.99% 的人不知道 AI 能做什么。Peter 给一个非技术朋友装了 OpenClaw。几天之内，朋友开始自己做小工具。一周之内，他升级到了 200 美元的订阅。

[第六篇](/you-are-the-manager-zh)说关键技能是管理 Agent，不是写代码。Peter 的工作流是极致版本：纯语音、不用 IDE、四到十个 Agent 并行。

[第七篇](/why-claude-code-zh)说缰绳比模型更重要。OpenClaw 从另一个方向证明了这一点——它可以用*任何*模型，因为缰绳才是创造 Agency 的关键。

[第八篇](/the-printing-press-moment-zh)说范式转移已经发生了。OpenClaw 是它正在扩展到代码之外的证明。Peter 用一小时做了它。它能自我重写。17.5万颗星。一个人，三个月，没有风投。

窗口还开着。但你已经知道了。
